# PI - Portability and Interoperability

## PI-01 Documentation and security of input and output interfaces
The cloud service can be accessed from other cloud services or cloud customers' IT systems via documented input and output interfaces. For this purpose, the interfaces' use for returning data is clearly documented for expert personnel.

Communication takes place via standardized communication protocols that ensure the confidentiality and integrity of the transmitted information in accordance with its protection requirements. Communication over untrusted networks is encrypted according to CRY-02.

The type and scope of the documentation for the interfaces are based on the needs of expert personnel of the cloud customers to enable the use of these interfaces. The information is maintained so that it is applicable to the provided cloud service in the version intended for productive use.

## PI-02 Contractual agreements for the provision of data
Contractual agreements define the following aspects regarding the termination of the contractual relationship, insofar as they are applicable to the cloud service:

- The type, scope, and format of the data that the cloud provider transfers to the cloud customer
- The definition of the period within which the cloud provider makes the data available to the cloud customer
- The definition of the point in time at which the cloud provider makes the data inaccessible to the cloud customer and deletes it
- The cloud customer's responsibilities and cooperation obligations in providing the data

The definitions are based on the needs of knowledgeable individuals of potential customers to assess the suitability of the cloud service with regard to their dependence on the cloud provider and legal and regulatory requirements.

## PI-03 Secure data deletion
The cloud provider's procedures for deleting the cloud customer's data upon termination of the contractual relationship ensure compliance with the contractual agreements (see PI-02).

Deletion includes data in the cloud customer's environment, metadata, and data in the data backup.

The deletion procedures prevent recovery by forensic means.
